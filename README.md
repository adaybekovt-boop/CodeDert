# CodeDert

CodeDert - это настольное приложение для работы с кодом вместе с AI. Его можно воспринимать как локальный рабочий стол для проекта: слева файлы, в центре редактор, справа чат с моделями, которые умеют читать проект, искать по файлам, объяснять код и вносить изменения через инструменты.

Главная идея CodeDert простая: открыть папку проекта и разговаривать с ней как с живым кодовым контекстом. Не копировать куски файлов в чат вручную, не держать в голове всю структуру проекта, не прыгать между десятком отдельных сервисов.

## Что это за приложение?

CodeDert помогает разбирать, писать и менять код внутри выбранной папки проекта.

В приложении есть:

- файловое дерево проекта;
- редактор кода;
- AI-чат с режимом работы по проекту;
- tool calls для чтения, поиска, создания и редактирования файлов;
- карта проекта, которая показывает структуру папок и файлов;
- Brain - локальная память/граф заметок о проекте;
- режимы для ревью, объяснения, фиксов, планирования и UI/design-задач;
- отдельный режим Chat With Model для общения с моделями и генерации медиа;
- генерация изображений и видео через подключенные провайдеры, если у вас есть ключи.

CodeDert не заменяет IDE полностью, но закрывает важный слой: быстро дать модели реальный доступ к проекту и получить практическую помощь по файлам, а не общий совет.

## Как оно работает?

1. Вы открываете папку проекта.
2. CodeDert сканирует структуру файлов и строит карту проекта.
3. Вы выбираете модель: локальную через Ollama или API-модель через провайдера.
4. В чате задаете задачу обычным языком.
5. Если задаче нужен реальный контекст, агент вызывает инструменты:
   - смотрит структуру папок;
   - читает нужные файлы;
   - ищет строки по проекту;
   - создает новые файлы;
   - редактирует существующие файлы;
   - может запускать разрешенные команды.
6. Результаты инструментов возвращаются обратно модели, и она продолжает работу уже на фактах из проекта.

То есть модель не просто "угадывает", где что находится. Она может проверить проект инструментами и только потом отвечать или менять файлы.

## Как начать пользоваться?

Если вы обычный пользователь, исходники клонировать не нужно.

1. Откройте страницу Releases:
   https://github.com/adaybekovt-boop/CodeDert/releases
2. Скачайте последний `CodeDert-Setup-...exe`.
3. Установите приложение.
4. Откройте нужную папку проекта.
5. Выберите локальную модель или подключите API-ключ в Settings.

Для локальных моделей понадобится Ollama:

https://ollama.com

После установки Ollama можно скачать подходящую кодовую модель и выбрать ее в CodeDert.

## Какие API-ключи доступны?

CodeDert поддерживает локальные модели и несколько внешних провайдеров.

Для обычного chat/code режима доступны:

| Провайдер | Для чего |
|---|---|
| Anthropic (Claude) | Сильные code/reasoning модели, агентные задачи, длинный контекст |
| OpenRouter | Единый доступ к множеству моделей разных компаний |
| OpenAI | ChatGPT/GPT-модели и часть мультимодальных сценариев |
| Google Gemini | Gemini-модели через совместимый endpoint |
| Groq | Быстрые open-source модели через API |
| NVIDIA NIM | Модели с build.nvidia.com |
| xAI (Grok) | Grok chat/code модели |
| Moonshot (Kimi) | Kimi/Moonshot модели |
| DeepSeek | DeepSeek chat/code модели |
| Qwen (DashScope) | Qwen-модели через DashScope |
| Custom | Любой OpenAI-compatible endpoint |

Для медиа в Chat With Model доступны отдельные провайдеры:

| Тип | Провайдеры |
|---|---|
| Изображения | OpenAI, Gemini, xAI |
| Видео | OpenAI, Gemini |

Важно: не каждая модель из списка провайдера является chat-моделью. Image/video/embedding/audio/realtime модели не должны использоваться в обычном чате. CodeDert фильтрует такие модели из chat selector и показывает понятную ошибку, если старый выбор все еще сохранен в настройках.

## Это безопасно?

CodeDert сделан как local-first приложение, но безопасность зависит от того, какие модели и ключи вы подключаете.

Что важно:

- API-ключи хранятся в системном хранилище Windows через keytar, а не в файлах проекта.
- Ключ каждого провайдера отправляется только к endpoint этого провайдера.
- Локальные модели через Ollama работают на вашем компьютере.
- Внешние API-модели получают только тот контекст, который вы отправляете им через чат/инструменты.
- Агентные изменения файлов проходят через инструменты приложения, а не через скрытый произвольный доступ.
- У команд и терминала есть ограничения/denylist, чтобы снизить риск опасных действий.
- Медиа-генерация использует отдельные API endpoints и не смешивается с обычным chat endpoint.

Но честно: если вы подключаете внешний API и просите модель читать файлы проекта, содержимое этих файлов может уйти провайдеру модели. Для приватного кода используйте локальные модели или внимательно выбирайте, какие задачи отдавать API-моделям.

## Какие есть команды?

Команды вводятся прямо в чат.

| Команда | Что делает |
|---|---|
| `/help` | Показывает список команд |
| `/clear` | Очищает историю чата |
| `/stop` | Останавливает текущую AI-задачу |
| `/settings` | Открывает настройки |
| `/models` | Показывает доступные модели |
| `/model <name>` | Переключает активную модель |
| `/index` | Пересканирует проект |
| `/ask <question>` | Задать вопрос по проекту без явной просьбы менять файлы |
| `/edit <instruction>` | Попросить изменить выбранный/активный файл |
| `/fix <symptom>` | Найти и исправить проблему |
| `/review [target]` | Проверить код на баги и риски |
| `/explain [target]` | Объяснить файл, функцию или ошибку |
| `/test [target]` | Предложить тесты |
| `/commit` | Предложить текст commit message |
| `/plan <task>` | Планирование задачи с последующим исполнением |
| `/multyplan <task>` | Многошаговый план: планирование, критика, подтверждение, исполнение |
| `/multyplan-approve` | Подтвердить последний план `/multyplan` |
| `/multyplan-reject` | Отклонить последний план `/multyplan` |
| `/ultrathink <task>` | Глубокий анализ проекта без изменений |
| `/design` | Режим UX/UI-критика |
| `/cdesign <brief>` | Генератор cinematic landing page |
| `/image <prompt>` | Генерация изображения |
| `/brain ...` | Работа с Brain-памятью проекта |

## Что вообще есть кроме кода?

CodeDert - это не только чат "напиши функцию". Внутри есть несколько рабочих зон.

### Карта проекта

Project Map показывает структуру проекта как интерактивную карту. Она помогает понять, где лежат основные папки и файлы, раскрывать ветки и быстро ориентироваться в большом проекте.

Если проект слишком большой, карта не молча прячет файлы: она показывает, где часть узлов скрыта лимитом.

### Brain

Brain - это локальная память проекта. Туда можно складывать важные заметки, решения, правила и наблюдения. Потом CodeDert может использовать эти заметки как дополнительный контекст.

Примеры:

- "В этом проекте API-роуты лежат в worker/index.js";
- "Перед релизом всегда запускать npm test";
- "Не менять дизайн-кнопки без проверки мобильной версии";
- "Вот почему мы выбрали такую модель/архитектуру".

### Chat With Model

Отдельная панель для общения с моделями и задач, где важны вложения, медиа и обычный диалог. Там же доступны image/video провайдеры, если подключены ключи.

### Design и CDesign

`/design` включает режим UI/UX-критика: можно просить модель смотреть на интерфейс, структуру страницы, текст, поведение и визуальную иерархию.

`/cdesign` предназначен для создания выразительных landing pages и frontend-направлений. Он сильнее ориентирован на визуальный результат, motion, композицию и анти-generic подход.

### Планирование

`/plan`, `/multyplan` и `/ultrathink` нужны для более крупных задач, когда простой "исправь файл" недостаточен.

- `/plan` разбивает задачу на шаги;
- `/multyplan` добавляет отдельный этап критики и подтверждения;
- `/ultrathink` анализирует проект глубже, но не редактирует файлы сам.

## Что умеют tools?

В code-режиме модель может вызывать инструменты приложения. Типичный набор:

- посмотреть структуру папки;
- прочитать файл;
- найти строку или TODO по проекту;
- заменить фрагмент в файле;
- создать новый файл;
- удалить файл;
- выполнить разрешенную команду;
- получить результат и продолжить рассуждение.

В интерфейсе такие действия видны отдельными tool-событиями, чтобы было понятно, что модель реально сделала, а не просто написала "я сейчас прочитаю файл".

## Для кого это?

CodeDert особенно полезен, если вы:

- работаете с локальными моделями и не хотите постоянно копировать код в браузер;
- хотите подключать разные API-модели в одном интерфейсе;
- часто просите AI читать и править файлы проекта;
- хотите видеть, какие инструменты модель реально вызвала;
- работаете с проектом, где важно сохранять контекст и решения;
- хотите совмещать код, дизайн, планирование и генерацию медиа в одном приложении.

## Ограничения

- Приложение в первую очередь ориентировано на Windows.
- Качество работы сильно зависит от выбранной модели.
- Локальные модели могут быть медленными и требовательными к VRAM/RAM.
- Внешние API могут стоить денег и получать часть контекста проекта.
- Некоторые модели провайдеров не являются chat-моделями и не подходят для code/tools режима.
- Очень большие проекты сканируются с лимитами, чтобы приложение оставалось отзывчивым.

## Коротко

CodeDert - это AI-рабочее место для проекта: открыть папку, выбрать модель, спросить, проверить, изменить, сохранить контекст и продолжить работу.

Он не обещает магию. Он дает модели руки внутри вашего проекта - и показывает, когда эти руки действительно что-то сделали.
